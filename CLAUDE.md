## Project Overview

**Vivero Don Juan** — a WhatsApp chatbot for a plant nursery. Customers can ask about plant care (sun, water, growth), get prices, and request quotes (presupuestos). All conversation is in Spanish.

## Architecture

```
WhatsApp customer
      ↓ (message)
   Kapso
      ↓ POST /webhook
   FastAPI (Railway)
      ↓ embed query (OpenAI text-embedding-3-small)
   Supabase pgvector  ←→  descripcion + catalogo tables
      ↓ top-3 plants + catalog variants
   Claude API (claude-sonnet-4-6)
      ↓ Spanish reply
   Kapso → WhatsApp customer
```

## Stack

- **Backend**: Python 3, FastAPI, uvicorn — `app/` package
- **AI**: OpenAI `text-embedding-3-small` for embeddings, Anthropic `claude-sonnet-4-6` for chat
- **Database**: Supabase (PostgreSQL + pgvector extension)
- **Deployment**: Railway — auto-deploys from `main` branch via `Procfile`
- **WhatsApp**: Kapso (Enterprise API)
- **Schema management**: Supabase GitHub integration — migrations in `supabase/migrations/` auto-deploy on push to `main`

## Data Model

**`descripcion`** table — plant master records, one row per species/cultivar
- `plant_id`, `nombre_cientifico`, `nombre_comun`, `categoria`, `exposicion_solar`, `riego`, `crecimiento`, `facilidad_cultivo`, `uso_recomendado`, `resistencia_frio`, `descripcion`, `embedding vector(1536)`, `created_at`

**`catalogo`** table — product variants (many per plant, different container sizes)
- `variante_id`, `plant_id` (FK), `nombre_comun`, `nombre_cientifico`, `tamano_envase_lts`, `precio`, `stock`, `elegibilidad`

Prices in ARS. `elegibilidad` rated 1–4. Search uses HNSW index with cosine distance.

2. Create a phone-number-scoped webhook (NOT project-level) pointing to the Railway URL:
   ```bash
   node scripts/create.js \
     --phone-number-id <PHONE_NUMBER_ID> \
     --url https://vivero-don-juan-production.up.railway.app/webhook \
     --events whatsapp.message.received \
     --payload-version v2
   ```
   The skill scripts live in `C:\Users\juan_kolibri\.claude\skills\integrate-whatsapp\scripts\`.

3. Copy the `webhook_secret_key` from the response and set it in Railway:
   - Railway dashboard → service → Variables → add `KAPSO_WEBHOOK_SECRET=<value>`
   - Railway will redeploy automatically.

4. Send a test WhatsApp message to the nursery number and verify the bot replies.

**Webhook payload notes (v2 format):**
- Incoming text: `payload["message"]["text"]["body"]`
- Sender phone: `payload["conversation"]["phone_number"]` (E.164 with `+`, stripped before sending)
- Our phone number ID: `payload["phone_number_id"]`
- Signature header: `X-Webhook-Signature` (HMAC-SHA256 hex of raw body)
- Event header: `X-Webhook-Event` — only `whatsapp.message.received` with `type == "text"` is processed
